# Highlight-to-AI 开发规划稿

## 1. 项目定位

`Highlight-to-AI` 是一款轻量级桌面文本处理工具：用户在任意应用中选中“自然语言指令 + 待处理文本”，按下全局快捷键后，工具自动复制选区、调用 OpenAI 兼容大模型处理，并将结果就地替换回原位置。

核心体验目标：

- **任意应用可用**：邮件、浏览器、笔记、聊天窗口、Office/WPS、代码编辑器等。
- **一键触发**：无需复制粘贴到 AI 对话框。
- **极速响应**：常见短文本目标 1~3 秒完成。
- **输出干净**：只替换为最终结果，不出现解释话术。
- **可配置**：支持自定义 API Key、Base URL、模型、快捷键、语言与剪贴板恢复策略。

## 2. 产品范围

### 2.1 MVP 范围

MVP 聚焦 Windows 优先落地，完成从选中文本到就地替换的完整闭环。

必做功能：

1. **全局快捷键触发**
   - 默认快捷键建议：`F3`。
   - 支持在配置中修改。
   - 处理过程中防重复触发。

2. **选中文本获取**
   - 触发后保存旧剪贴板内容。
   - 模拟 `Ctrl + C` 复制当前选区。
   - 等待剪贴板更新并读取文本。
   - 空文本或复制失败时提示并中止。

3. **OpenAI 兼容 API 调用**
   - 支持配置：`base_url`、`api_key`、`model`、`timeout`、`max_tokens`、`temperature`。
   - 使用固定系统提示词，要求模型只输出最终结果。
   - API 超时、鉴权失败、网络错误时给出反馈。

4. **就地替换**
   - 将模型返回文本写入剪贴板。
   - 模拟 `Ctrl + V` 覆盖原选区。
   - 默认恢复旧剪贴板，可配置关闭。

5. **基础状态反馈**
   - Windows 使用 `winsound` 提供开始、成功、失败提示音。
   - 失败时不进行粘贴。

6. **配置文件**
   - 本地 `config.toml` 管理基础配置。
   - API Key 不提交到仓库。
   - 首次启动时自动生成示例配置。

### 2.2 Beta 范围

在 MVP 稳定后补齐可用性能力。

- 系统托盘入口：状态、设置、退出。
- 简易设置窗口：编辑 API、模型、快捷键、语言、Prompt 模板。
- API 连通性测试按钮。
- 桌面通知：处理中、成功、失败。
- 中英文界面文案。
- 日志文件与错误诊断。
- PyInstaller 打包为 Windows 可执行文件。

### 2.3 后续增强范围

- macOS 支持：`Cmd + C/V`、辅助功能权限引导、菜单栏应用。
- Prompt 模板库：润色、翻译、总结、扩写、改语气等。
- 多模型配置与快速切换。
- 历史记录与撤销辅助。
- 流式响应降低感知延迟。
- 原生 Win32 全局热键替代第三方监听库，提高稳定性。
- 开机自启。

## 3. 推荐技术路线

### 3.1 MVP 技术栈

| 能力 | 推荐方案 | 说明 |
|---|---|---|
| 运行语言 | Python 3.11+ | 内置 `tomllib`，生态成熟 |
| 全局热键 | `pynput` | 跨平台潜力更好，适合后续扩展 |
| 按键模拟 | `pyautogui` | 发送 `Ctrl+C` / `Ctrl+V` |
| 剪贴板 | `pyperclip` | 读写简单，支持等待剪贴板变化 |
| LLM 调用 | `openai` | 支持 OpenAI 兼容 `base_url` |
| 状态音 | `winsound` | Windows 标准库，MVP 零依赖 |
| 配置 | TOML | 人类可读，适合桌面工具 |
| 打包 | PyInstaller | 单文件 EXE 或目录模式 |

### 3.2 Beta 技术栈补充

| 能力 | 推荐方案 | 说明 |
|---|---|---|
| 托盘 | `pystray` | 轻量、跨平台 |
| 设置窗口 | `tkinter` 或 `customtkinter` | MVP 可先用内置 Tkinter，后续美化 |
| 通知 | Windows Toast 库或托盘通知 | 优先托盘通知，避免复杂集成 |
| 密钥存储 | `keyring` | 后续替代明文配置中的 API Key |

## 4. 里程碑规划

### M0：项目初始化与工程骨架

目标：建立可运行、可配置、可测试的基础工程。

交付物：

- `pyproject.toml` 或 `requirements.txt`。
- `src/` 代码目录。
- `config.example.toml`。
- `README.md` 更新：安装、配置、运行说明。
- `.gitignore` 忽略本地配置、日志、构建产物。

验收标准：

- 可以通过命令行启动程序。
- 缺失配置时能生成或提示示例配置。
- 不会把 API Key 写入仓库。

### M1：核心闭环原型

目标：跑通“选中文本 -> 快捷键 -> LLM -> 替换”。

交付物：

- 全局热键监听。
- 剪贴板保存、复制、读取、写入、恢复。
- OpenAI 兼容 API 客户端。
- 默认系统提示词。
- 基础提示音。

验收标准：

- 在记事本、浏览器输入框、邮件编辑器中选中文本后可成功替换。
- 空选区不会调用 API。
- API 失败不会粘贴错误内容。
- 处理期间重复按快捷键不会并发执行多次。

### M2：稳定性与错误处理

目标：降低误操作和失败成本。

交付物：

- 复制超时处理。
- API 超时处理。
- 剪贴板恢复策略。
- 任务状态锁。
- 日志记录。
- 成功、失败、取消的统一反馈。

验收标准：

- 网络断开、Key 错误、选区为空时均能安全失败。
- 失败后用户原剪贴板尽量保持不变。
- 日志中能定位失败原因。

### M3：托盘与配置界面

目标：降低普通用户使用门槛。

交付物：

- 系统托盘图标。
- 设置窗口。
- API 测试按钮。
- 快捷键配置。
- 中英文界面文案。
- 退出菜单。

验收标准：

- 用户无需编辑配置文件即可完成常用设置。
- 修改配置后重启可生效；可选支持立即生效。
- 托盘可正常退出后台进程。

### M4：打包与发布

目标：形成可分发版本。

交付物：

- PyInstaller 打包脚本。
- Windows 发布包。
- 用户安装/运行说明。
- 常见问题文档。

验收标准：

- 新 Windows 环境可按文档运行。
- 配置文件位置明确。
- 打包版本可正常监听热键、调用 API、替换文本。

## 5. 开发优先级

| 优先级 | 功能 | 原因 |
|---|---|---|
| P0 | 热键、复制、LLM、粘贴闭环 | 项目核心价值 |
| P0 | API 配置 | 没有配置无法真实使用 |
| P0 | 错误处理与剪贴板恢复 | 直接影响安全感 |
| P1 | 提示音/通知 | 无 UI 工具必须有反馈 |
| P1 | 托盘退出 | 防止后台进程难以管理 |
| P1 | 设置窗口 | 降低使用门槛 |
| P2 | Prompt 模板 | 提升效率但非核心闭环 |
| P2 | 打包安装器 | 发布阶段需要 |
| P3 | 跨平台 | MVP 先 Windows 优先 |

## 6. 配置设计草案

建议使用 `config.toml`：

```toml
[app]
language = "zh-CN"
restore_clipboard = true
beep = true
show_notification = false

[hotkey]
trigger = "<f3>"

[api]
base_url = "https://api.deepseek.com"
api_key = ""
model = "deepseek-chat"
timeout_seconds = 15
max_tokens = 2048
temperature = 0.3

[prompt]
system = "你是一个极简的文本处理引擎。用户会给你一段文本，其中包含处理指令和待处理内容。请直接执行指令，并只输出最终处理结果，不要解释，不要寒暄。"
```

后续可将 `api_key` 迁移到系统密钥库，配置文件只保存引用。

## 7. 测试计划

### 7.1 功能测试

- 选中中文文本，要求润色。
- 选中英文文本，要求改写。
- 选中“指令 + 内容”，确认输出只包含结果。
- 空选区触发。
- API Key 错误。
- Base URL 错误。
- 网络超时。
- 快捷键连续触发。
- 处理完成后剪贴板是否恢复。

### 7.2 应用兼容测试

优先测试：

- Windows 记事本。
- 浏览器输入框。
- Web 邮箱。
- 微信/企业微信输入框。
- Word/WPS。
- VS Code 文本编辑器。

### 7.3 体验测试

- 100 字以内短文本：目标 1~3 秒。
- 500 字以内中等文本：目标 3~8 秒。
- 开始处理有明确反馈。
- 成功替换有明确反馈。
- 失败时不会覆盖用户文本。

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 用户切换焦点导致误粘贴 | 高 | 提示音/通知、缩短耗时、失败不粘贴、后续增加焦点校验 |
| 热键冲突 | 中 | 支持自定义热键，启动时提示注册失败 |
| 剪贴板被其他应用抢占 | 中 | 记录触发前后剪贴板变化，超时中止 |
| 模型输出废话 | 中 | 强系统 Prompt，后续增加轻量清洗 |
| API 延迟过高 | 高 | 推荐低 TTFT 模型，配置超时，后续支持流式 |
| API Key 明文保存 | 中 | MVP 提醒风险，Beta 引入 `keyring` |
| 第三方库兼容问题 | 中 | 封装接口，后续可替换 Win32 原生实现 |

## 10. MVP 验收定义

当以下条件全部满足，可认为 MVP 完成：

1. 用户可配置 OpenAI 兼容 API URL、Key、模型与快捷键。
2. 用户在任意常见文本输入应用中选中“指令 + 内容”后，可以一键替换为 AI 结果。
3. 失败场景不会破坏原文，不会错误粘贴空内容。
4. 处理过程有基础可感知反馈。
5. 默认恢复用户原剪贴板。
6. 项目可按 README 在 Windows 上安装并运行。
